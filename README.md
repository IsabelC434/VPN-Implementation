# Secure Remote Access Implementation Using Client to Site VPN

## Project Summary
#### This project demonstrates implementation and validation of secure remote access using a client to site Virtual Private Network (VPN) within a Windows Server environment.
#### The setup includes configuring a VPN server using Routing and Remote Access Service (RRAS), enabling user authentication, and establishing a secure connection from a client machine. The project also includes testing connectivity to internal network resources and troubleshooting common VPN-related issues.

  - ### Technologies and Tools:
    - Microsoft Azure (Virtual Machine Hosting)
    - Windows Server 2022 (VPN Server/Domain Controller)
    - Windows 10 Client Machine
    - Routing and Remote Access Service (RRAS)
    - Active Directory Domain Services (AD DS)
    - Remote Desktop (RDP)
    - PowerShell (ping, ipconfig, nslookup)
  - ### Objectives
    - Configure a client to site VPN using RRAS
    - Enable and manage VPN access for domain users
    - Establish secure remote connectivity froam a client machine
    - Validate access to internal network sources over VPN
    - Troubleshoot common VPN connectivity and authentication issues
  - ### Environment setup
    - Hosting environment: Microsoft Azure
    - Domain controller/VPN Server: Windows Server 2022
    - Client machine: Windows 10
    - Domain: isabelsdomain.com
      - ### Administrative tools
        - RRAS
        - ADUC
        - Powershell
